1. Polimorfisme Overriding

Polimorfisme overriding adalah kemampuan class anak untuk menimpa atau mengganti method yang ada pada class induk sehingga method yang dijalankan adalah method dari class anak.

class Hewan { void suara() { print("Hewan bersuara"); } } class Kucing extends Hewan { @override void suara() { print("Kucing mengeong"); } } void main() { Hewan h = Kucing(); h.suara(); }

2. Named Constructor

Named constructor adalah constructor yang memiliki nama khusus sehingga dalam satu class bisa memiliki lebih dari satu constructor.

class Mahasiswa { String nama; int umur; Mahasiswa(this.nama, this.umur); Mahasiswa.lulus(this.nama){ umur = 22; } } void main() { Mahasiswa m = Mahasiswa.lulus("Yoza"); print(m.nama); print(m.umur); }

3. Abstraksi

Abstraksi adalah proses menyembunyikan detail implementasi dan hanya menampilkan fungsi penting kepada pengguna.

abstract class Kendaraan { void jalan(); } class Mobil extends Kendaraan { void jalan() { print("Mobil berjalan"); } } void main() { Mobil m = Mobil(); m.jalan(); }

4. Interface

Interface adalah class yang hanya berisi method tanpa isi dan harus diimplementasikan oleh class lain.

abstract class Bentuk { void gambar(); } class Lingkaran implements Bentuk { void gambar() { print("Gambar Lingkaran"); } } void main() { Lingkaran l = Lingkaran(); l.gambar(); }

5. Inheritance

Inheritance adalah pewarisan sifat dari class induk ke class anak sehingga class anak dapat menggunakan variabel dan method dari class induk.

class Hewan { String nama = "Kucing"; } class Kucing extends Hewan { void tampil() { print(nama); } } void main() { Kucing k = Kucing(); k.tampil(); }

6. Overloading

Overloading adalah pembuatan method dengan nama yang sama tetapi parameter yang berbeda.

class Hitung { int tambah(int a, int b, [int c = 0]) { return a + b + c; } } void main() { Hitung h = Hitung(); print(h.tambah(2,3)); print(h.tambah(2,3,4)); }

7. Encapsulation

Encapsulation adalah teknik menyembunyikan data dengan menggunakan getter dan setter.

class Mahasiswa { String _nama; void setNama(String nama){ _nama = nama; } String getNama(){ return _nama; } } void main() { Mahasiswa m = Mahasiswa(); m.setNama("Yoza"); print(m.getNama()); }

8. Abstract Class

Abstract class adalah class yang tidak bisa dibuat objek dan digunakan sebagai dasar untuk class lain.

abstract class Hewan { void suara(); } class Sapi extends Hewan { void suara(){ print("Moo"); } } void main() { Sapi s = Sapi(); s.suara(); }

9. Polimorfisme

Polimorfisme adalah kemampuan objek untuk memiliki banyak bentuk atau perilaku yang berbeda.

class Hewan { void suara() { print("Hewan bersuara"); } } class Ayam extends Hewan { void suara() { print("Kukuruyuk"); } } void main() { Hewan h = Ayam(); h.suara(); }

10. Static

Static digunakan untuk membuat method atau variabel yang bisa dipanggil tanpa membuat objek.

class Matematika { static int tambah(int a, int b){ return a + b; } } void main() { print(Matematika.tambah(5,3)); }

11. Class dan Object

Class adalah blueprint atau rancangan, sedangkan object adalah hasil dari class tersebut.

class Buku { String judul; Buku(this.judul); } void main() { Buku buku1 = Buku("Pemrograman"); print(buku1.judul); }

12. Constructor

Constructor adalah method yang otomatis dipanggil saat objek dibuat.

class Mahasiswa { String nama; Mahasiswa(this.nama); } void main() { Mahasiswa m = Mahasiswa("Yoza"); print(m.nama); }

13. This Keyword

This digunakan untuk menunjuk variabel dalam class agar tidak tertukar dengan parameter.

class Mahasiswa { String nama; Mahasiswa(String nama){ this.nama = nama; } } void main() { Mahasiswa m = Mahasiswa("Yoza"); print(m.nama); }

14. Program Luas Persegi

Program ini menggunakan konsep class dan method untuk menghitung luas persegi.

class Persegi { int sisi; Persegi(this.sisi); int luas(){ return sisi * sisi; } } void main() { Persegi p = Persegi(4); print(p.luas()); }

Kesimpulan

Dari praktikum ini dapat disimpulkan bahwa OOP memiliki banyak konsep seperti Polimorfisme, Abstraksi, Inheritance, Encapsulation, Interface, Constructor, Static, dan This Keyword. Dengan OOP program menjadi lebih terstruktur, mudah dikembangkan, dan dapat digunakan kembali.
